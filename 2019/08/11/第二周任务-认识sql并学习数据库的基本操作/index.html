几种数据库

关系型数据库

我的理解就是一层一层的递进，依稀记得数据库老师说的第一范式、第二范式，还有一堆包裹同心圆什么的。就是说数据库效率要高其中的字段相互的关系必须达到最简单。这样更改一个数据库其中的字段不会导致大范围的修改数据库。关系型数据库就是有各种表构成的，可以使用SQL结构化查询语言对数据库进行操作的数据库类型。我们常见的数据表都是关系型数据库的标志。它的优点是易于维护、使用方便、可以进行复杂操作。缺点就是对于大型的数据读写效率比较低，高并发读写要求对于硬盘I/O是一个很大的瓶颈。

代表的关系型数据库有MySQL、Oracle、SQL Server

非关系型数据库

非关系型数据库我没有使用过，也是上网查资料发现原来总是在耳边环绕的Redis、MongoDB都是非关系型数据库。非关系型数据库严格来说不是一种数据库，应该是一种数据结构话存储方式的集合，可以是文档或者键值对等等。非关系型数据库的优点是格式灵活、速度快、扩展性高；缺点是不支持sql语言、无法做复杂查询。

代表的非关系型数据库有NoSQL、Reids、MongoDB
